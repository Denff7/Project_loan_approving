## Довідник: Займ vs. Кредит

Для кращого розуміння контексту, що використовується в проєкті, наводимо порівняння фінансових продуктів:

| Параметр порівняння | Займ (Позика) | Кредит (Споживчий) |
| :--- | :--- | :--- |
| **Сторони угоди** | Фізичні та юридичні особи, підприємці, МФО, ломбарди. | Кредитор - **тільки банк**, позичальник - необмежене коло осіб. |
| **Предмет договору** | Гроші чи **матеріальні цінності** (речі). | Тільки гроші. |
| **Вид угоди** | Письмове чи **усне**. | **Тільки письмове**. |
| **Зобов'язання** | Тільки у позичальника. | У кредитора та позичальника. |
| **Спосіб повернення** | Декількома платежами або **однією сумою**. | Регулярні виплати **за графіком** (ануїтетні платежі). |
| **Процентна ставка** | Не обмежена: від $0$ до $100\%$ річних та вище. | Залежить від ставки банку, більш регламентована. |

### Методологія Підготовки Даних (Views.py)

Вся підготовка даних та тренування моделі інтегровані в єдиний **Pipeline** (Конвеєр) для забезпечення чистоти коду та **уникнення витоку даних** (Data Leakage).

1.  **Кодування категоріальних змінних:**
    * Використовується **One-Hot Encoding (OHE)** для номінальних ознак (`Married`, `Education`, `Property_Area` тощо). Це перетворює категорії (наприклад, 'Graduate', 'Not Graduate') на бінарні стовпці (0 та 1), що є необхідним для ML-алгоритмів.
2.  **Нормалізація числових змінних:**
    * Використовується **StandardScaler** для числових ознак (`ApplicantIncome`, `LoanAmount` тощо). Це приводить їх до **середнього значення 0 та стандартного відхилення 1**, запобігаючи домінуванню ознак із більшим масштабом.
3.  **Модель:**
    * Використовується **RandomForestClassifier** - це універсальний і популярний алгоритм машинного навчання, який використовується для задач класифікації (прогнозування категорії, наприклад, схвалено/відмовлено).
